```markdown
# Daloopa API 数据格式规范

## 接口：get_financials()

### 返回格式
- 数据格式: JSON
- 数字: Float，2位小数
  - 示例: 1234567.89 (不带$符号)
  - 精度: 小数点后2位
- 日期: String, YYYY-MM-DD
  - 示例: "2023-12-31"
- 缺失值: null (不是NaN，不是""，是 null)
- 货币: 统一USD，返回中无符号

### 在Datapack Builder中的应用
- 接收: revenue = 1234567.89 (来自JSON)
- 应用: RULE 1 (财务数据→$格式)
- 转换: 百万级 → $1,234.6M
- 记录: [Daloopa] [get_financials] [AAPL]

### 常见问题
Q: 为什么返回值没有小数点？
A: 单位是美元，整数表示。需要除以1000000获得百万。
```

```

#### 改进5：跨源数据合并规范

**改进方案：**
```markdown
# 当需要合并多个MCP源的数据时

## 原则
1. **明确主源** (primary source)
   - 如果用Daloopa Revenue，就坚持用它
   - 其他源作为验证，不作为主要数据来源

2. **记录转换过程**
   - 来源A: 1000 (千为单位)
   - 转换: 1000 / 1 = 1000 (仍是千)
   - 来源B: 1 (百万为单位)
   - 转换后比对: 1000千 = 1百万 ✓ 一致

3. **保留源引用**
   - [Daloopa] [get_financials]
   - [AKShare] [stock_financial_analysis_indicator]

## 冲突处理
如果两个源数据不一致：
```
1. 单位是否相同? (例如都是百万?)
2. 时期是否相同? (都是2023FY?)
3. 如果都相同但数据不同，记录差异
4. 在Datapack脚注中说明使用了哪个源
```
